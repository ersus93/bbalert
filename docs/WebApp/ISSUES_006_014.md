# 💬 Issue #008 — Envío de Mensajes desde el Panel

## 📋 Descripción

Añadir la funcionalidad de enviar mensajes de Telegram directamente desde el panel web, tanto a un usuario individual como a todos los usuarios (broadcast).

---

## 🎯 Objetivos

- Modal de "Enviar mensaje" en la vista de detalle de usuario
- Sección de "Mensaje Masivo" en el panel con editor de texto y preview
- Soporte para formato HTML básico (negrita, cursiva, links)
- Indicador de progreso para el broadcast masivo
- Límite respetado: máximo 30 mensajes/segundo (Telegram rate limit)

---

## 💻 Implementación Backend

```python
# routers/users.py

class MessageRequest(BaseModel):
    text: str
    parse_mode: str = "HTML"

@router.post("/{user_id}/message")
async def send_message(
    user_id: str,
    req: MessageRequest,
    admin=Depends(get_current_admin)
):
    result = await bot_sender.send_message(user_id, req.text, req.parse_mode)
    return result

@router.post("/broadcast")
async def broadcast(
    req: MessageRequest,
    background_tasks: BackgroundTasks,
    admin=Depends(get_current_admin)
):
    users = data_reader.get_users()
    background_tasks.add_task(send_broadcast, list(users.keys()), req.text)
    return {"status": "started", "total": len(users)}

async def send_broadcast(user_ids: list, text: str):
    for uid in user_ids:
        await bot_sender.send_message(uid, text)
        await asyncio.sleep(0.05)  # 20 msg/s, bajo el límite de 30
```

---

## ✅ Criterios de Aceptación

- [ ] El mensaje individual llega al usuario de Telegram en menos de 3 segundos
- [ ] El broadcast masivo se ejecuta en background y no bloquea la UI
- [ ] El panel muestra una barra de progreso del broadcast en tiempo real (polling al endpoint de estado)
- [ ] Se respeta el rate limit de Telegram (delay entre mensajes)
- [ ] Mensajes vacíos o que excedan 4096 caracteres devuelven error 422
- [ ] Los errores de envío (usuario bloqueó el bot) se registran pero no detienen el broadcast

# 🌐 Issue #013 — Despliegue nginx + SSL

**Fase**: 6 — Despliegue  
**Prioridad**: 🔴 Alta

**Rama**: `feature/webapp-fase-6-deploy`  
**Depende de**: #001 — #012 completadas

---

## 📋 Descripción

Configurar **nginx** como reverse proxy delante de la aplicación FastAPI (uvicorn) y generar un certificado SSL con **Let's Encrypt** (certbot) para que el panel sea accesible de forma segura por HTTPS.

---

## 🎯 Objetivos

- Configurar un virtualhost nginx para el dominio/subdominio del panel
- Proxy de peticiones HTTPS → uvicorn en localhost:8080
- Certificado SSL automático con certbot
- Forzar redirección HTTP → HTTPS
- Configurar cabeceras de seguridad (HSTS, X-Frame-Options, etc.)

---

## 📄 Configuración nginx

```nginx
server {
    listen 80;
    server_name panel.tudominio.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name panel.tudominio.com;

    ssl_certificate /etc/letsencrypt/live/panel.tudominio.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/panel.tudominio.com/privkey.pem;

    # Cabeceras de seguridad
    add_header Strict-Transport-Security "max-age=31536000" always;
    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 📄 Comandos de Despliegue

```bash
# Instalar certbot
sudo apt install certbot python3-certbot-nginx

# Generar certificado
sudo certbot --nginx -d panel.tudominio.com

# Recargar nginx
sudo nginx -t && sudo systemctl reload nginx
```
